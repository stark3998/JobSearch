## 8. Resume Generation Instructions

### 8.1 File Setup

The resume uses a LaTeX template. Always start from `Base/cv.tex`.

```bash
# Copy base template
cp Base/cv.tex Resumes/cv_<company>.tex

# Create empty citations file (required by biblatex)
touch citations.bib

# Compile
pdflatex -interaction=nonstopmode cv_<company>.tex
```

**Required LaTeX packages** (install if missing):
```bash
apt-get install -y texlive-latex-extra texlive-fonts-recommended texlive-fonts-extra texlive-bibtex-extra biber
```

**Known compile issues to watch for:**
- Escape characters: run `sed -i 's/\x1b//g' cv_<company>.tex` before compiling
- `\hfill` in `\section{}` headings causes hyperref errors — always wrap with `\texorpdfstring{...}{}`
- Duplicate `\end{document}` from bash `echo` appends — check the tail of the file
- Missing `citations.bib` — always `touch citations.bib` before compiling

### 8.4 Bullet Writing Rules

1. **Lead with the action verb** — Built, Engineered, Developed, Conducted, Implemented, Designed, Architected, Prototyped, Hardened, Led
2. **Bold the technology and outcome** — `\textbf{Azure Kubernetes Service}`, `\textbf{60\%}`
3. **Include a metric on every bullet** — reduction %, user scale, application count, time savings
4. **Mirror JD language** — if JD says "supply chain attacks", the bullet should say "supply chain attacks" not "pipeline security"
5. **One idea per bullet** — don't combine two unrelated achievements
6. **Offensive security roles (Google):** follow the loop — discover vulnerability → build PoC → ship systemic fix
7. **Architecture roles (Starbucks):** frame outputs as deliverables — "produced formal risk assessment", "developed reusable reference architecture"
8. **Engineering roles (Anthropic):** emphasize ownership — "from identifying the gap to shipping and monitoring the fix"
9. **DevOps roles (Costco):** emphasize operational metrics — SLA, cost reduction, uptime, onboarding velocity
